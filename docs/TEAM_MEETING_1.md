# 第一次团队会议议程

**时间**：第1周周末或第2周初
**地点**：线上/线下
**时长**：30-45分钟

---

## 会议目标
1. 统一对项目的理解
2. 确定技术栈
3. 明确第一周任务
4. 建立协作流程

---

## 议程

### 1. 项目概述（5分钟 - 苏崇博）
- 介绍项目目标和PDF内容
- 展示已搭建的目录结构
- 说明9周时间线

### 2. 技术栈决策（10分钟 - 全员讨论）
**需要决定的问题**：
- [ ] 前端用什么？（纯JS / Vue / React）
- [ ] 是否用虚拟环境？
- [ ] 数据存储格式？（JSON确定，CSV可选）

**投票方式**：
- 王俊涛介绍前端选项优劣
- 团队举手投票
- 当场确定

### 3. 环境配置（10分钟 - 侯钧译）
**现场操作**：
```bash
# 每个人都执行
git clone <仓库地址>
cd Housing-Factor-Mining
python -m venv venv
venv\Scripts\activate  # Windows
pip install -r requirements.txt
```

如果有问题当场解决

### 4. Git协作规范（10分钟 - 苏崇博）
- 分支策略（见下文）
- 提交规范
- 代码审查流程

### 5. 第一周任务分配（5分钟）
- 确认每人具体任务
- 约定下次检查时间（3-5天后）
